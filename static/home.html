What is DevSuite?

Everything you
reach for daily,
in one place.

A locally-hosted, offline-first developer suite that runs entirely on your machine. No subscriptions, no sign-ups, no data leaving your network — ever.

Features

Built for developers
who mean it.

DevSuite doesn't cut corners. Every tool is built with the quality and depth a professional workflow demands.

Monaco-powered Diff

Same engine as VS Code. Side-by-side and inline views, 20+ language syntax highlighting, merge arrows, and .patch export.

AES-256 Secret Vault

KeePass-style local vault. AES-256-GCM encryption with BLAKE2b integrity for passwords, tokens, and API credentials. Never leaves disk.

SSH Terminal & SFTP

Multi-tab SSH client with integrated SFTP file browser. Key and password auth, encrypted profiles, WSL discovery.

JSON / YAML Linting

Real-time validation, pretty-print, minify, key sorting, YAML → JSON. Supports K8s, Docker Compose, GitHub Actions configs.

Cron Visualizer

Build and validate cron expressions for Unix, Quartz, AWS EventBridge, GitHub Actions. 28-day fire heatmap and human-readable descriptions.

Local API Tester

Privacy-first REST client. Saved collections, offline history, CORS proxy — zero telemetry, runs entirely on your machine.

Privacy, by design.

Your data stays
exactly where it belongs.

DevSuite was built from day one with one rule — nothing leaves your machine. Not your code. Not your secrets. Not your usage patterns.

No cloud, ever

Every tool runs entirely in your browser or on localhost. No servers receive your data. No uploads. No API calls to external services.

No accounts needed

No sign-up, no login, no license key, no email. Open the page and start working. DevSuite respects your time and your privacy.

AES-256 encrypted

Where data is stored — Secret Vault, DevDB Manager — it is protected with AES-256-GCM encryption and BLAKE2b integrity checks. Your secrets stay yours.

A note from the author

The honest truth about DevSuite

DevSuite started as a weekend hobby project — because my wife, a developer herself, kept struggling without a decent local diff tool, a way to format code, or a safe place to lint without uploading them to some random cloud service. So naturally, the most reasonable response was to spend my weekends building an entire developer suite. As you do.

Spoiler: she uses it every day. Husband points secured. Mission accomplished.

That said — this is still a hobby project, built by one person, in spare time. It may have bugs. Some edges may be rough. A feature might occasionally misbehave in a way that makes you wonder if I tested it at all (I did, I promise — just... optimistically). If you hit something broken, please don't suffer in silence.
