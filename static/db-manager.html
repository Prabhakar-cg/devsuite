🛡️ Security note: The Vault and SSH Profiles stores contain client-side AES-256-GCM encrypted blobs — the server never decrypts or inspects their contents, even when DevDB itself is unencrypted.

Every .dsb file starts with a fixed 64-byte header. In plaintext mode the payload is prefixed with a 32-byte BLAKE2b-256 checksum. In encrypted mode the AESGCM output (ciphertext ‖ 16-byte GCM auth tag) follows the header.

🔑 Set Database Password

Setting a password enables AES-256-GCM server-side encryption for the entire database file. Leave both fields blank to switch to integrity-only mode (BLAKE2b, no encryption).

ℹ️ A server restart is required for the password change to take effect. The password must be provided to DevDB at startup (e.g., via environment variable).
